Installation af openxds-repository

https://svn.nspop.dk/svn/nsp/trunk/docker/components/openxds/compose/release/docker-compose.yml

Variable, der skal rettes til - ansible eller manuelt:
 - OPENXDS_SERVICE_URL =  Load-balancer-URL der peger på <openxds repository docker>:8020/axis2/services/xdsrepositoryb (92_insert_POC_repository.sql)
 - OPENXDS_UNIQUE_REPOSITORY_OID = Repository oid, e.g. 1.3.6.1.4.1.21367.2010.1.2.1125 - use OID of current opentext-repository for proof of concept (compose)
 - OPENXDS_COMMUNITY_OID = home community, e.g. urn:oid:1.3.6.1.4.1.21367.2010.1.2.2045 (compose)
 - OPENXDS_REPOSITORY_PORT = External docker-port to repository (compose)
 - XDS_LOG_LEVEL = DEBUG (compose)
 - OPENXDS_DATADB_URL = jdbc-url til data-db (compose)
 - OPENXDS_DATADB_USER =  brugernavn til data-db (compose)
 - OPENXDS_DATADB_PASS = password til data-db (compose)
 - OPENXDS_LOGDB_URL = jdbc-url til log-db (compose)
 - OPENXDS_LOGDB_USER =  brugernavn til log-db (compose)
 - OPENXDS_LOGDB_PASS = password til log-db (compose)
 - OPENTEXT_REGISTRY_HOST = registry name, e.g. test1-cnsp.ekstern-test.nspop.dk (compose)
 - OPENTEXT_REGISTRY_PORT = registry host, e.g. 8080 (compose)
 - OPENTEXT_REGISTRY_PATH = registry path, e.g. /registry/services/xds-iti42 (compose)
 
Log-volume-mapping ændres efter behov

Containeren har brug for to mariadb-10.1-databaser, der initialiseres med
  * data-db: 
     - https://svn.nspop.dk/svn/nsp/trunk/docker/components/openxds/compose/database/repository-data-db/01_openxds.sql
    
  * log-db:
     - https://svn.nspop.dk/svn/nsp/trunk/docker/components/openxds/compose/database/log-db/01_openxdslogs.sql


Derudover skal DDS have kendskab til det nye repository, så følgende skal udføres på dds'ens database:
   - https://svn.nspop.dk/svn/nsp/trunk/docker/components/openxds/compose/database/dds-db/92_insert_POC_repository.sql
Den eksisterende række i tabellen documentsources.documentsource med samme oid, der peger på opentext-repository'et skal slettes
